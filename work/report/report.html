S09

1) Elaborate on a few (3-5) different security issues or concerns, from your own experience, related to web application security and relate them to the OWASP top ten.

1- Login Authentication: showing user’s username and password on network tab when sending the request to the backend! It makes this possibility for hackers to find user’s credential and they can reveal easily user’s personal data (which is against GDPR). This problem is mentioned in OWASP as "A3-Sensitive Data Exposure".

2. SQL injection: being able to login to the web page without having a valid password! It means that we only check if the username is right and we do not check the password. It means that everyone who has the username, can get logged in and have access to user’s personal data (which is against GDPR).This is also the first on the list of OWASP. According to OWASP this problem can happen in any kind of system such as a web program written in PHP while sending data to MySQL database, when the program is transferring the user input to a mail server, and etc.

3. Websites should create a cookie session and session ID for each valid session. These cookies usually carry important data like username, password and probably user’s personal information. When the session is ended (when the user gets logged out or timeout or browser closed abruptly), these cookies should be invalidated. If they do not get invalidated, the data will stay in the system and hackers can have access to the user’s personal data easily by checking the cookies. This problem is also mentioned in the list as "A2-Broken Authentication".

2) Summarize by providing your own guidelines to a new web programmer on the topic “The essential guidelines to follow to create secure web applications”.

As users are trusting to us by proving their personal information, then we are responsible to keep their information hidden in all ways. Our highest priority should be providing a safe web app for users. We can easily make that by multi-factoring authentications. It means that instead of asking users to type their passwords, we can use other ways like face-recognition. And we can add timeout session for making cookies invalid!

In this part I have learned about the common weaknesses and vulnerabilities that hackers can use to misuse and access websites, how they usually happen and what is the solution to fix them.I also have learned about GDPR (General Data Protection Regulation) framework and some of it guidelines.
